## Overview

This repository contains all code for the CH248 project on modelling non-additive effects in the liquid‚Äìliquid phase separation (LLPS) of the RNA-binding protein FUS. The project develops a minimal three-component extended Flory‚ÄìHuggins (FH) free energy framework for FUS, PEG (crowder), and water, where ionic strength enters as an external control parameter through salt-dependent interaction parameters.

The central question is whether the effects of salt and crowder on FUS phase separation are strictly additive, or whether a non-trivial coupling term `Œ¥` is required. The framework provides a falsifiable, quantitative route to answering this from experimental phase diagram data.

---

## Model

The free energy density of the ternary system is:

$$f = \frac{\phi_p}{N_p}\ln\phi_p + \frac{\phi_c}{N_c}\ln\phi_c + \phi_s\ln\phi_s + \chi_\mathrm{eff}(I)\,\phi_p^2 + \big(K_1\phi_c + \delta I\phi_c\big)\phi_p$$

with ionic-strength-dependent parameters:

$$\chi_\mathrm{eff}(I) = \chi_0 - A\sqrt{I} + BI \qquad \chi_{pc}(\phi_c, I) = K_1 + \delta I$$

Phase boundaries are located using the **spinodal condition** ‚Äî the Hessian determinant of $f$ vanishing:

$$\det(H) = H_{pp}H_{cc} - H_{pc}^2 = 0$$

**Free parameters:** $\chi_0$, $A$, $B$, $K_1$, $\delta$ (five; $K_2$ was found negligible and fixed to zero).

**Default true parameters (synthetic data):**

| Parameter | Value | Description |
|-----------|-------|-------------|
| $N_p$ | 600 | FUS chain length |
| $N_c$ | 227 | PEG chain length |
| $\chi_0$ | ‚àí12.0 | Bare protein‚Äìsolvent interaction |
| $A$ | ‚àí32.0 | Debye‚ÄìH√ºckel screening coefficient |
| $B$ | ‚àí18.0 | Ion-correlation coefficient |
| $K_1$ | ‚àí5.0 | Crowder‚Äìprotein coupling |
| $\delta$ | ‚àí1.0 | **Salt‚Äìcrowder coupling (parameter under test)** |

## Usage

**Run in order:**

### 1. Generate synthetic data
```bash
python generate_synthetic.py
```
Generates 15,000 labelled phase-separation data points with 3% label noise across two stages:
- **Stage 1:** $(\phi_p, I)$ at $\phi_c \approx 0$, across low ($I \in [0.001, 0.5]$) and high ($I \in [1.0, 3.0]$) ionic strength ranges
- **Stage 2:** $(\phi_p, \phi_c)$ at fixed $I \in \{0.1, 0.25, 0.4\}$ M

Edit `TRUE_PARAMS` at the top of the file to explore different parameter regimes. Set `delta = 0.0` to generate null-hypothesis data.

### 2. Sequential fitting (recommended)
```bash
python fit_spinodal_1.py
# or with custom data directory and chain lengths:
python fit_spinodal_1.py --data-dir path/to/data --Np 600 --Nc 227
```
Two-stage fitting pipeline:
- **Stage 1:** fits $\chi_0$, $A$, $B$ from $\phi_c = 0$ data using logistic NLL minimisation with 300 L-BFGS-B restarts (differential evolution warm start)
- **Stage 2:** fixes Stage-1 parameters, fits $K_1$ and $\delta$ from the multi-$I$ crowder data
- Performs a **likelihood ratio test** (LRT) for $H_0: \delta = 0$ vs $H_1: \delta \neq 0$, with AIC/BIC comparison and a 95% profile likelihood CI for $\delta$

### 3. Joint fitting (comparison)
```bash
python joint_fit.py
# or:
python joint_fit.py --data-dir path/to/data --Np 600 --Nc 227
```
Fits all five parameters ($\chi_0$, $A$, $B$, $K_1$, $\delta$) simultaneously from Stage-2 data only. Less robust than sequential fitting ‚Äî there is an identifiability problem between $B$ and $\delta$ when only low-$I$ data are used, leading to up to 77% relative error on $\delta$ (vs 0.83% for sequential).
